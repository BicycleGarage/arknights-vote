## 🎨 图片存储方案详解

### 为什么图片存储很重要？

你的项目有13个干员，每个干员2-4张立绘：
- 总计约 30-50 张图片
- 每张 300-500KB
- 总共约 10-25MB

如果直接放在GitHub：
- ❌ 仓库体积大
- ❌ 加载速度慢
- ❌ GitHub流量限制

### 方案对比

| 方案 | 速度 | 稳定性 | 成本 | 难度 |
|------|------|--------|------|------|
| **阿里云OSS+CDN** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ¥5-15/月 | ⭐⭐ |
| jsDelivr+GitHub | ⭐⭐⭐ | ⭐⭐⭐⭐ | ¥0 | ⭐ |
| Cloudflare R2 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 前10GB免费 | ⭐⭐ |
| Firebase Storage | ⭐⭐⭐ | ⭐⭐⭐⭐ | 免费额度内 | ⭐⭐ |

### 🏆 推荐：阿里云OSS + CDN

**原因**:
1. 你已经有阿里云账号
2. 国内速度最快
3. 成本极低（几块钱一个月）
4. 配置简单

**配置步骤**:
1. 创建OSS Bucket（公共读）
2. 上传所有图片
3. 开启CDN加速（可选）
4. 修改 `config.js` 中的 `IMAGE_CDN_BASE`

详见 [`ALIYUN_DEPLOYMENT.md`](ALIYUN_DEPLOYMENT.md) 方案C

## 🔧 快速开始步骤

### 如果你想立即开始（30分钟）

#### 第1步：Firebase设置（15分钟）
1. 访问 [Firebase Console](https://console.firebase.google.com/)
2. 创建项目 `arknights-vote`
3. 创建Realtime Database（测试模式）
4. 复制配置信息

#### 第2步：配置代码（5分钟）
编辑 `config.js`:
```javascript
const firebaseConfig = {
  apiKey: "你的API_KEY",
  // ... 其他配置
};

const IMAGE_CDN_BASE = 'https://cdn.jsdelivr.net/gh/你的用户名/arknights-vote@main/images';
```

#### 第3步：部署到GitHub（10分钟）
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/arknights-vote.git
git push -u origin main
```

然后在GitHub Settings → Pages 启用。

**完成！** 你现在有一个可以多人投票的系统了。

---

### 如果你想优化（额外30分钟）

#### 第4步：配置阿里云OSS
1. 创建Bucket
2. 上传图片
3. 修改 `config.js`:
```javascript
const IMAGE_CDN_BASE = 'https://你的bucket.oss-cn-hangzhou.aliyuncs.com/images';
```

#### 第5步：配置自定义域名
1. DNS添加CNAME: `vote` → `你的用户名.github.io`
2. GitHub Pages设置自定义域名
3. 启用HTTPS

**完成！** 现在你有专业的域名和快速的图片加载。
